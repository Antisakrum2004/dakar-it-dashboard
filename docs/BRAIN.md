## 📌 1. О проекте

**dakar-it-dashboard** — статичный HTML-дашборд для отдела ИТ Дакар/АтиЛаб. Не Next.js, не React, не сборщик. Один файл `index.html` (~4400 строк), чистый HTML/CSS/JS, деплойится на GitHub Pages.

**Репозиторий:** https://github.com/Antisakrum2004/dakar-it-dashboard
**Прод:** https://antisakrum2004.github.io/dakar-it-dashboard/
**Структура:**
```
/index.html          — всё приложение в одном файле
/.github/workflows/deploy.yml  — автодеплой на GitHub Pages при push в main
/docs/BRAIN.md       — этот файл (память проекта)
/skills/             — AI skills (не относятся к разработке, остались от форка)
/download/           — артефакты (пусто)
```

**Стек:** HTML5 + CSS3 (в `<style>`) + vanilla JS (ES6, async/await). Никаких npm-зависимостей, кроме CDN: `exceljs` + `FileSaver` для экспорта в Excel.

**Деплой:** push в `main` → GitHub Actions → GitHub Pages. Время деплоя: 30-60 секунд.

## 🎯 4. Логика вкладки Внедрение (главная фича)

### 4.1. Концепция
Управление очередью внедрения задач 1С в демо/рабочую базу через **теги Bitrix24** (stateless, без своей БД). Источник истины — теги задачи в Битрикс24.

### 4.2. Машина состояний (6 тегов, кириллица)

| Тег | Назначение |
|-----|------------|
| `внедрить_демо` | Пользователь поставил — задача ждёт внедрения в ДЕМО |
| `в_чате_демо` | Технический маркер: алерт в чат уже отправлен (защита от дублей) |
| `внедрена_в_демо` | Задача внедрена в ДЕМО (тумблер нажат) |
| `внедрить_раб` | Аналогично для РАБОЧЕЙ базы |
| `в_чате_раб` | Аналогично для РАБОЧЕЙ базы |
| `внедрена_в_рабочую` | Аналогично для РАБОЧЕЙ базы |

### 4.3. Цикл жизни задачи

```
[Пользователь ставит тег внедрить_демо]
        ↓
[Сайт: кнопка "Обновить"]
   - Опрос tasks.task.list за 3 дня с filter[TAG]=внедрить_демо
   - Если у задачи НЕТ в_чате_демо:
     • im.message.add в chat7568: "⚠ Внедрить в ДЕМО [URL]..."
     • tasks.task.update — добавить в_чате_демо
        ↓
[Задача видна в активной секции "На внедрение в ДЕМО"]
   - Карточка с тумблером "В демо" справа
        ↓
[Внедряющий нажал тумблер]
   - tasks.task.update: снять внедрить_демо + в_чате_демо, добавить внедрена_в_демо
   - im.message.add + im.message.update(URL_PREVIEW=N): 
     "[S]✅ Задача [URL] «название» [B]внедрена в ДЕМО базу 19.06 17:05[/S][/B]"
   - Сохранение в localStorage (VN_DONE_LOG_KEY) — для показа "✅ Внедрена DD.MM HH:MM"
        ↓
[Задача видна в секции "✅ Внедрено в ДЕМО"]
   - Голая зачёркнутая строка с URL + кнопка "↩ Вернуть"
        ↓
[Пользователь нажал "↩ Вернуть"]
   - tasks.task.update: снять внедрена_в_демо, добавить внедрить_демо + в_чате_демо
   - im.message.add + update(URL_PREVIEW=N): "↩ Задача возвращена во внедрение в ДЕМО базу..."
   - Задача снова в активной секции
```

### 4.4. Polling — кнопка «Обновить»

Алгоритм `loadVnedrenieData()` (строка ~3230):
1. `fetchVnedrenieUsers()` — получить всех пользователей (для имён)
2. Вычислить дату 3 дня назад (`VN_DAYS_BACK = 3`)
3. **4 последовательных запроса** `tasks.task.list` с `filter[>=CHANGED_DATE]` + `filter[TAG]=...`:
   - `внедрить_демо` → активные в демо
   - `внедрить_раб` → активные в рабочей
   - `внедрена_в_демо` → выполненные в демо
   - `внедрена_в_рабочую` → выполненные в рабочей
4. Дедупликация по task.id
5. Определение состояния (`rab_new`, `rab_queued`, `rab_done`, `demo_new`, `demo_queued`, `demo_done`)
6. Для `*_new` (нет маркера в_чате_*): отправить алерт в чат + добавить маркер
7. Сортировка: `rab_new` → `rab_queued` → `demo_new` → `demo_queued` → `rab_done` → `demo_done`

**Почему 4 запроса, а не 1?** Bitrix24 не поддерживает OR-логику через `filter[TAG][]` (проверено — возвращает 0). Нужно отдельный запрос на каждый тег.

### 4.5. Отправка сообщений в чат — КРИТИЧНО

**Проблема:** Bitrix24 при отправке URL в чат автоматически прикрепляет **rich-preview** — большую карточку с названием задачи, постановщиком, дедлайном, описанием. Это ломает формат.

**Решение** (функция `vnSendChatMessage(text, noAttach)`, строка ~3146):
- `noAttach=false` (по умолчанию): простой `im.message.add` → Bitrix24 сам прикрепляет rich-preview. **Используется для ⚠ алертов** (нужно показать карточку задачи).
- `noAttach=true`: `add` + сразу `im.message.update` с параметром `URL_PREVIEW: 'N'` на верхнем уровне (НЕ внутри `PARAMS`) → карточка удаляется, остаётся только текст с BB-кодами. **Используется для ✅ отбивок и ↩ возвратов**.

### 4.6. BB-коды в чате Bitrix24

Подтверждено работает (проверено тестами):
- `[S]...[/S]` — зачёркнутый текст
- `[B]...[/B]` — жирный
- `[URL]https://...[/URL]` — кликабельная ссылка (URL внутри)
- `[URL=https://...]текст[/URL]` — ссылка с кастомным текстом

**Формат ✅ отбивки** (точно как просил пользователь):
```
[S]✅ Задача [URL]https://1c-cms.bitrix24.ru/.../view/7810/[/URL] «ТЕСТ: Загрузка прайса» [B]внедрена в РАБОЧУЮ базу 19.06 17:05[/S][/B]
```
Вся строка зачёркнута, жирная часть тоже зачёркнута (т.к. `[S]` обёртывает `[B]`).

### 4.7. UI вкладки Внедрение

**3 секции** (управляются фильтрами в тулбаре):
1. «На внедрение в РАБОЧУЮ базу» — красный счётчик, grid 2 колонки, карточки с тумблером «В рабочую»
2. «На внедрение в ДЕМО базу» — синий счётчик, grid 2 колонки, карточки с тумблером «В демо»
3. «✅ Внедрено» — серая компактная секция, 1 колонка, голые зачёркнутые строки + кнопка ↩ Вернуть

**Активная карточка** (полный размер):
```
┌──────────────────────────────────────────────────────────┐
│ 🧪 ТЕСТ: Доработка отчёта Валовая прибыль                │
│ [Дакар] [🧪 Демо] [📅 24.06] Андрей Предеин [💬 3]       │
│                                       [тумблер] В демо   │
└──────────────────────────────────────────────────────────┘
```

**Выполненная задача** (голая строка, без карточки):
```
✅ Задача https://1c-cms.bitrix24.ru/.../view/7808/ «ТЕСТ: Сверка данных...» внедрена в ДЕМО базу 19.06 17:05    [↩ Вернуть]
```
Вся строка зачёркнута, URL — кликабельная ссылка, кнопка «↩ Вернуть» справа.

### 4.8. Превью задачи (открывается по клику на активную карточку)

Переиспользует `.preview-panel` из Дакара. Содержит:
- Заголовок, проект, бейдж (⚡ Рабочая / 🧪 Демо)
- Постановщик, исполнитель, дедлайн, дата создания, описание

**Важно:** поле ввода времени УБРАНО (по запросу пользователя — лишнее).

## 🐛 5. История ошибок и их решения

### 5.1. TAGS в tasks.task.update принимает ТОЛЬКО массив строк
**Симптом:** `TAGS: "crm"` (строка) → silent fail, тег не добавляется.
**Решение:** `TAGS: ["crm"]` (массив строк) → работает.
**Дата:** 19.06.2026, обнаружено при тестах API.

### 5.2. filter[TAG][] для OR-логики не работает
**Симптом:** `filter[TAG][]=tag1&filter[TAG][]=tag2` → возвращает 0 задач.
**Решение:** делать отдельный запрос на каждый тег (4 запроса для Внедрения).
**Дата:** 19.06.2026.

### 5.3. tasks.task.list возвращает tags в разном формате
**Симптом:** иногда `{tagId: {id, title}}` (объект), иногда `[{id, title}]` (массив).
**Решение:** функция `vnExtractTagTitles(tagsField)` обрабатывает оба формата.
**Дата:** 19.06.2026.

### 5.4. task.commentitem.getlist НЕ возвращает прикреплённые файлы
**Симптом:** в ответе нет полей `FILES`, `UF_FORUM_MESSAGE_DOC`, только текстовые.
**Решение:** файлы .cfe/.epf не показываются в превью напрямую. Даём ссылку «Открыть в Битрикс24» — внедряющий качает там.
**Дата:** 19.06.2026.

### 5.5. Дубликаты ✅ сообщений в чате
**Симптом:** пользователь кликал тумблер 3 раза → 3 идентичных ✅ в чате.
**Решение:** 
- В `vnOnInlineToggleDone`: проверка `toggle.classList.contains('done')` и `toggle.classList.contains('saving')` в начале
- В `vnOnToggleDone`: проверка `checkbox.disabled` и `task.state === '*_done'`
- После успешного клика: `checkbox.disabled = true` + `toggle.style.pointerEvents = 'none'`
**Дата:** 19.06.2026.

### 5.6. Rich-preview карточки в чате ломали формат
**Симптом:** при отправке URL в чат Bitrix24 автоматически прикреплял большую карточку с задачей (название, постановщик, дедлайн, описание). Занимала пол-экрана.
**Решение:** после `im.message.add` вызвать `im.message.update` с `URL_PREVIEW: 'N'` на верхнем уровне.
**Дата:** 19.06.2026.

### 5.7. URL_PREVIEW=N не работает в PARAMS
**Симптом:** `PARAMS: {"URL_PREVIEW": "N"}` → не убирает карточку.
**Решение:** передавать на верхнем уровне JSON: `{MESSAGE_ID, MESSAGE, URL_PREVIEW: 'N'}`.
**Дата:** 19.06.2026.

### 5.8. Все сообщения в чате получались без карточек (пользователь недоволен)
**Симптом:** пользователь хотел, чтобы ⚠ алерты были С rich-preview (полноценные карточки), а ✅ отбивки — БЕЗ (просто зачёркнутый текст). Я же убирал attach для всех.
**Решение:** добавил параметр `noAttach` в `vnSendChatMessage(text, noAttach)`. ⚠ алерты вызываются без второго аргумента (с attach), ✅ и ↩ — с `true` (без attach).
**Дата:** 19.06.2026.

### 5.9. Bitrix24 не даёт физически удалить «Сообщение удалено»
**Симптом:** после `im.message.delete` в чате остаётся надпись «Это сообщение было удалено» — её нельзя ни отредактировать, ни удалить повторно (`CANT_EDIT_MESSAGE`).
**Решение:** отправить «разделитель» — новое сообщение, которое отодвигает удалённые вниз в видимой области чата. Полной очистки истории нет.
**Дата:** 20.06.2026.

### 5.10. Список Дакара просачивался на вкладку Внедрение
**Симптом:** на вкладке Внедрение внизу отображались задачи Дакара (CRM-смарт-процесс).
**Решение:** добавил класс `.dakar-only` к `<div class="container">` (контейнер списка Дакара).
**Дата:** 19.06.2026.

### 5.11. Карточки Внедрения занимали всю ширину
**Симптом:** при большом количестве задач список растягивался неудобно.
**Решение:** CSS grid `.vnedrenie-section-body { grid-template-columns: 1fr 1fr; }` — 2 колонки на широких экранах, 1 колонка на узких (<900px).
**Дата:** 19.06.2026.

### 5.12. Выполненные задачи отображались как карточки (лишний размер)
**Симптом:** пользователь хотел, чтобы выполненные были компактными голыми строками, а не карточками.
**Решение:** в `vnRenderTaskCard` — если `isDone`, рендерим просто `<div class="vnedrenie-done-line">` без обёртки `.vnedrenie-task-item`. CSS — font-size 12px, text-decoration: line-through, без фона/бордера.
**Дата:** 19.06.2026.

### 5.13. Тумблер в превью — поле времени было лишним
**Симптом:** пользователь сказал «Время есть которое можно изменять в отрывшейся задаче - это лишнее».
**Решение:** убрал `<input type="number" id="vnTimeInput">` из `openVnedreniePreview`. Также убрал вызов `vnAddElapsedTime` (учёт времени не пишется автоматически).
**Дата:** 19.06.2026.

## ⚠️ 11. Известные ограничения

1. **Bitrix24 не удаляет «Сообщение удалено»** — после `im.message.delete` в истории остаётся надпись. Нет API для физической очистки.
2. **Нет файлов .cfe/.epf в превью** — `task.commentitem.getlist` не возвращает ID прикреплённых файлов. Нужен обходной путь через `disk.attachedObject.get` (медленно). Сейчас даём ссылку «Открыть в Битрикс24».
3. **Polling за 3 дня** — если задача была изменена >3 дней назад, она не попадёт в выборку. Контролируется константой `VN_DAYS_BACK = 3`.
4. **Webhook 116 не админ чата** — `im.chat.get` возвращает null. Не может изменять чужие сообщения в чате.
5. **Нет авторизации** — любой, кто знает URL, может нажать тумблер. Вопрос безопасности открыт.
